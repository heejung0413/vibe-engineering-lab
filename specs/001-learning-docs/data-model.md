# Data Model: 바이브코딩 및 개발 학습 문서

**Feature**: 001-learning-docs
**Date**: 2026-02-07

## 엔티티 정의

### 학습 문서 (Learning Document)

마크다운 파일로 저장되는 학습 콘텐츠의 논리적 구조

| 필드 | 설명 | 예시 |
|------|------|------|
| 제목 | 문서의 주제를 나타내는 H1 헤딩 | `# Spec Kit 소개` |
| 난이도 | 입문/중급/고급 | `난이도: 입문` |
| 학습 목표 | 이 문서를 통해 배울 내용 | 목표 목록 |
| 사전 지식 | 필요한 배경 지식 | 링크 또는 설명 |
| 개념 설명 | 핵심 개념에 대한 설명 | 텍스트 + 다이어그램 |
| 실습 | 따라할 수 있는 예제 | 코드 블록 + 명령어 |
| 정리 | 핵심 내용 요약 | 요약 목록 |
| 관련 문서 | 연관된 다른 학습 문서 링크 | 상대 경로 링크 |
| 작성일 | 문서 최초 작성일 | `2026-02-07` |
| 수정일 | 마지막 수정일 | `2026-02-07` |

### 폴더 카테고리 (Folder Category)

문서를 분류하는 최상위 디렉토리

| 카테고리 | 경로 | 용도 |
|----------|------|------|
| 바이브코딩 기술 | `Vibe-Coding/` | 바이브코딩 세션에서 얻은 기술/팁 |
| 개발 학습 | `개발-학습/` | 일반적인 개발 지식 |

### 하위 폴더 (Subfolder)

특정 도구나 주제에 대한 문서를 그룹화

| 상위 폴더 | 하위 폴더 | 용도 |
|-----------|-----------|------|
| Vibe-Coding | speckit | Spec Kit 도구 관련 문서 |
| Vibe-Coding | claude-code | Claude Code 사용법 |
| 개발-학습 | python | Python 학습 문서 |
| 개발-학습 | typescript | TypeScript 학습 문서 |

## 문서 구조 템플릿

```markdown
# [제목]

> 난이도: [입문/중급/고급]
> 작성일: YYYY-MM-DD
> 수정일: YYYY-MM-DD

## 학습 목표

이 문서를 통해 다음을 배울 수 있습니다:
- 목표 1
- 목표 2

## 사전 지식

- [필요한 사전 지식 또는 관련 문서 링크]

## 개념 설명

[핵심 개념에 대한 설명]

## 실습

### 단계 1: [제목]

[설명 및 코드 예제]

### 단계 2: [제목]

[설명 및 코드 예제]

## 정리

- 핵심 포인트 1
- 핵심 포인트 2

## 관련 문서

- [관련 문서 1](../path/to/doc1.md)
- [관련 문서 2](../path/to/doc2.md)
```
